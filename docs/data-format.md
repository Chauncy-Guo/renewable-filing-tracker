# 数据格式说明

## 1. 公告 API 响应

```
POST https://sso.renewable.org.cn/api/tyrz/uca/notice/list2
{
    "pageNo": 1,
    "pageSize": 50,
    "noticeType": 1,
    "noticeProjectCategory": 1
}
```

返回示例:

```json
{
    "errorCode": 0,
    "data": [
        {
            "noticeTitle": "26年4月份全国新增建档立卡新能源发电项目情况(不含户用光伏)",
            "noticeUrl": "https://sso.renewable.org.cn/noticeHisroty/detail/xxxxx",
            "noticeState": 1,
            "noticeTime": "2026-05-15 10:00:00",
            "noticeProjectCategory": 1
        }
    ]
}
```

`query_api.js` 过滤条件:

- `noticeState == 1` (已发布)
- 标题包含 **"不含户用光伏"** 或 **"全国新增建档立卡新能源发电项目情况"**
- 年月在用户给定范围内

输出 `output/api_records_target.json` 结构:

```json
[
    { "year": 2026, "month": 4, "title": "...", "url": "https://...", "time": "..." }
]
```

## 2. 输入文件 (output/PDFs/)

### 全国新能源装机 Word

- 文件名: `YY年M月份全国新能源装机建档立卡数据.{doc,docx}`
- 24 年 5~8 月: 含 4 个表 (风机 / 集中式光伏 / 工商业分布式 / 生物质)
- 25 年 3 月起: 只含 3 个表 (风机 / 集中式光伏 / 生物质)

每个表头固定为:

| 序号 | 项目编码 | 项目名称 | 装机容量(MW) | 项目所在地 |
| --- | --- | --- | --- | --- |
| 1 | PWC.... | ... | 50 | 内蒙古自治区... |

### 工商业分布式光伏 PDF

- 文件名: `YY年M月份工商业分布式光伏建档立卡数据.pdf`
- 24 年 9 月起按月发布
- 表格无项目名称列, 仅: 序号 / 项目编码 / 装机容量(MW) / 项目所在地
- 解析时仅填 4 列, 名称留空

## 3. 中间产物 (output/Excel/)

每个 .docx / .pdf 解析后生成一个 xlsx:

```
YY年M月份{类型}建档立卡数据.xlsx
```

> 类型取值: **工商业分布式 / 集中式光伏 / 风机 / 生物质**

标准表头 (5 列):

| 序号 | 项目编码 | 项目名称 | 装机容量(MW) | 项目所在地 |
| --- | --- | --- | --- | --- |

## 4. 汇总 (output/汇总/)

4 个文件, 每行都附加"项目月份"列:

| 序号 | 项目编码 | 项目名称 | 装机容量(MW) | 项目所在地 | 项目月份 |
| --- | --- | --- | --- | --- | --- |
| 1 | PWC... | 某风电场 | 50.0 | 内蒙古自治区通辽市... | 2025年3月份 |
| 2 | PPC... | 某光伏电站 | 100.0 | 河北省张家口市... | 2025年3月份 |

排序规则: **同类型内按月份从最晚到最早**; 各文件内"项目月份"列保持原顺序, 方便回溯原始公告。

## 5. 字段清洗规则

| 字段 | 清洗动作 |
| --- | --- |
| 序号 | 必须是 1~4 位纯数字; 否则整行丢弃 |
| 项目编码 | 匹配 `^[A-Z]{2,3}\d{8,}[A-Z0-9]*$`; 否则丢弃 |
| 装机容量 | 浮点数; 4 位数字且在 2020-2039 视为年份(被排除) |
| 项目名称 / 项目所在地 | 跨多 token 拼接 (PDF) / 跨多单元格拼接 (Word) |
